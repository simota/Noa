## L0 â€” Vision
- **Problem**: Noa's About panel (macOS-standard `orderFrontStandardAboutPanel`) is implemented, but its content is thin â€” only `CARGO_PKG_VERSION`. There is no embedded git hash or build timestamp (no build.rs), and since `ApplicationIcon` is unspecified, icon display is not guaranteed for non-bundled launches (`cargo run`).
- **Target audience**: users who use or develop Noa as a macOS app (primarily the developer themself).
- **Job-to-be-done**: complete the app's polish as a macOS application â€” the About panel should show accurate version + build info and the app icon + name.
- **Definition of success**: the About panel correctly displays version, build info, icon, and name (for both bundled and non-bundled launches).

## L1 â€” Requirements

### Functional Requirements

| ID | Requirement |
|---|---|
| R-1 | The About panel's `ApplicationIcon` is resolved via a two-stage lookup: (1) when launched from a bundle, `noa.icns` under `NSBundle.mainBundle().resourcePath`; (2) when launched unbundled (`cargo run`), falls back to `../../assets/noa.icns` relative to `CARGO_MANIFEST_DIR`. |
| R-2 | If neither (1) nor (2) has a file present, `show_about_macos()` must not panic or crash â€” it falls back to the AppKit standard icon display by simply not setting the `ApplicationIcon` key. |
| R-3 | Add a new `crates/noa-app/build.rs` that embeds a short git hash (7 hex digits, equivalent to `git rev-parse --short=7 HEAD`) at build time as the `NOA_GIT_HASH` environment variable via `cargo:rustc-env=`. |
| R-4 | The same build.rs embeds the UTC build date (`YYYY-MM-DD`) as the `NOA_BUILD_DATE` environment variable. If `SOURCE_DATE_EPOCH` is set, it takes priority (for reproducible builds); otherwise, the UTC date at build time is used. |
| R-5 | If git is unavailable (outside a repository, git not installed, etc. â€” e.g. a build from an extracted tarball), build.rs embeds `NOA_GIT_HASH` as an empty string (`env!()` must always have a value emitted so the build stays compilable). `NOA_BUILD_DATE` is always obtained independently of git availability (R-4). When `NOA_GIT_HASH` is an empty string, the About version display falls back to the bare `CARGO_PKG_VERSION` only. |
| R-6 | When the git hash and date are both non-empty, the About panel's `ApplicationVersion` is displayed in the form `"{CARGO_PKG_VERSION} ({NOA_GIT_HASH}, {NOA_BUILD_DATE})"` (e.g. `"0.1.0 (a1b2c3d, 2026-07-05)"`). This extended display is limited to the About panel. |
| R-7 | The explicit icon specification (R-1, R-2) and the build-metadata embedding (R-3â€“R-6) are implemented as two independent commits (A = icon, B = build metadata) so each can be reverted individually. |

## L2 â€” Detail (Development)

### Component design

**A. Explicit icon specification** â€” `crates/noa-app/src/app_actions.rs`
- The path-selection logic is factored into a test-injectable pure helper: `icon_path_from_candidates(candidates: &[PathBuf]) -> Option<PathBuf>` â€” checks `Path::exists()` in order from the front and returns the first path that exists. Returns `None` if no candidate exists (per R-2, the `ApplicationIcon` key is not added to the options dictionary).
- `show_about_macos()` builds the candidate list and passes it in:
  1. `NSBundle::mainBundle()` â†’ `resourcePath` (NSString) â†’ `{resourcePath}/noa.icns` (for bundled launches; matches the file placed by `bundle-macos.sh:41-42`).
  2. `Path::new(env!("CARGO_MANIFEST_DIR")).join("../../assets/noa.icns")` (for non-bundled launches such as `cargo run`; references `assets/noa.icns` at the workspace root directly).
- When the helper returns `Some(path)`, an `NSImage` is created via `initWithContentsOfFile:` and set on the `options` dictionary under the `ApplicationIcon` key (using the same objc2 `setObject:forKey:` pattern as the existing `ApplicationName`/`ApplicationVersion`).

**B. build.rs build metadata** â€” new `crates/noa-app/build.rs`
- Runs `std::process::Command::new("git").args(["rev-parse", "--short=7", "HEAD"])` from `CARGO_MANIFEST_DIR`. On success (exit 0 and non-empty stdout), emits the trimmed stdout as `NOA_GIT_HASH`; on failure (not a git repository, git not installed, error exit), emits an empty string.
- Date: if the `SOURCE_DATE_EPOCH` environment variable is set, it is converted to UTC `YYYY-MM-DD`; otherwise, the output of `std::process::Command::new("date").args(["-u", "+%Y-%m-%d"])` is used and emitted as `NOA_BUILD_DATE` (independent of git availability â€” date embedding always succeeds).
- Always emits: `println!("cargo:rustc-env=NOA_GIT_HASH={hash}")` / `println!("cargo:rustc-env=NOA_BUILD_DATE={date}")` (emitted even when the value is an empty string, guaranteeing that the consumer-side `env!("NOA_GIT_HASH")` always compiles â€” R-5).
- In addition to `println!("cargo:rerun-if-changed=build.rs")`, emits rerun triggers pointing at the repository's `.git/HEAD`, `.git/refs` (a directory â€” cargo watches it recursively), and `.git/packed-refs` (concrete paths such as `CARGO_MANIFEST_DIR/../../.git/HEAD`, etc. â€” noa-app lives at `crates/noa-app`, and the workspace-root `.git` is two levels up).
- Adds a `version_string() -> String` helper on the `app_actions.rs` side: if both `NOA_GIT_HASH` and `NOA_BUILD_DATE` are non-empty, returns `format!("{} ({}, {})", CARGO_PKG_VERSION, hash, date)`; if either is empty, returns the bare `CARGO_PKG_VERSION` (R-5, R-6). `show_about_macos()`'s `ApplicationVersion` uses this `version_string()`. `cli.rs:138` / `terminal.rs:1527` continue to reference `env!("CARGO_PKG_VERSION")` directly and are unchanged (NFR-2).

### Commit split (R-7)
- Commit A: add `icon_path_from_candidates()` + `ApplicationIcon` setup logic (`app_actions.rs` only).
- Commit B: add `build.rs` + `version_string()` + swap in the `ApplicationVersion` call (`app_actions.rs` + `build.rs`).

## Scope

### In Scope
- Explicit `ApplicationIcon` specification for the About panel (bundled / non-bundled two-stage lookup).
- Embedding and displaying build metadata (7-digit git hash + UTC build date), limited to the About panel.
- Adding `crates/noa-app/build.rs`.

### Out of Scope
- A `-dirty` suffix (detecting/displaying uncommitted changes) â€” trimmed during stress-testing.
- Additional build metadata such as rustc version and target triple.
- Rich-text Credits (a list of rustc/major crate versions, Ghostty acknowledgments) â€” candidate C, rejected as over-decoration relative to how rarely this screen is viewed.
- Expanded CLI `+version`-equivalent detail output â€” candidate D, `noa --version` (`cli.rs:138`) already suffices and this avoids expanding scope into the apprt surface.
- Non-macOS About UI (remains log-only no-op).
- Adding any config key related to this feature.

## Considered but Rejected
- **C. Rich-text Credits**: an option to add a display of rustc/major crate versions + Ghostty acknowledgments. Rejected by the user's judgment as over-decoration for a rarely viewed screen.
- **D. Expanded CLI detailed version**: an option to add Ghostty `+version`-equivalent detailed output to the CLI. Rejected because the existing `noa --version` (`cli.rs:138`) already meets the requirement, and to avoid expanding scope into the apprt surface (CLI).
- **E. YAGNI baseline (stop at A only)**: a minimal option that finishes with just the explicit icon specification. Build metadata (B) directly serves the JTBD of strengthening About and its implementation cost is small, so the slightly broader A+B was adopted over the baseline.
